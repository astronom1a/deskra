# DK310 Feature Design Spec

**Date:** 2026-05-15
**Status:** Approved

---

## Overview

Halaman baru **DK310** untuk mengelola Daftar Perubahan Penambahan Kayu Jati (Model DK No. 310). User dapat mengimport file Excel (format DK310P), data tersimpan ke Supabase, dan dapat dilihat per periode maupun rincian per surat bukti dengan breakdown per mutu.

Dapat diakses oleh **operator** (data TPK sendiri) dan **admin** (semua TPK).

---

## Database Schema (Supabase)

### Tabel: `dk310_periods`
Satu baris per file Excel yang diimport.

| Kolom | Tipe | Keterangan |
|---|---|---|
| id | uuid PK | default gen_random_uuid() |
| tpk_id | uuid FK | references tpk(id), tenant scoping |
| periode | text | e.g. "I-2026" |
| masa_pembayaran | text | e.g. "I - Januari 2026" |
| kph | text | e.g. "KPH BANYUWANGI UTARA" |
| bkph | text | e.g. "BKPH WATU DODOL" |
| tanggal_cetak | date | |
| penambahan_btg | integer | |
| penambahan_m3 | numeric(10,3) | |
| sisa_lalu_btg | integer | |
| sisa_lalu_m3 | numeric(10,3) | |
| jumlah_persediaan_btg | integer | |
| jumlah_persediaan_m3 | numeric(10,3) | |
| jumlah_pengurangan_btg | integer | |
| jumlah_pengurangan_m3 | numeric(10,3) | |
| sisa_sekarang_btg | integer | |
| sisa_sekarang_m3 | numeric(10,3) | |
| created_at | timestamptz | default now() |
| created_by | uuid FK | references auth.users(id) |

### Tabel: `dk310_surat_bukti`
Satu baris per surat bukti per periode (biasanya 59 baris per file).

| Kolom | Tipe | Keterangan |
|---|---|---|
| id | uuid PK | |
| period_id | uuid FK | references dk310_periods(id) on delete cascade |
| urutan | integer | nomor urut dalam periode |
| jenis | text | e.g. "DKB(P51+)" |
| tanggal | text | e.g. "06" |
| nomor_surat | text | e.g. "262130220000059" |
| jumlah_total_btg | integer | total semua mutu |
| jumlah_total_m3 | numeric(10,3) | total semua mutu |

### Tabel: `dk310_surat_bukti_mutu`
Satu baris per mutu per surat bukti — inti normalisasi Opsi A.

| Kolom | Tipe | Keterangan |
|---|---|---|
| id | uuid PK | |
| surat_bukti_id | uuid FK | references dk310_surat_bukti(id) on delete cascade |
| kategori | text | 'KB' (Kayu Bernomor) atau 'TBN' (Tak Bernomor) |
| mutu_code | text | e.g. 'AIII_IS', 'AIII_PD', 'AI_M', dll |
| mutu_label | text | e.g. 'AIII IS/UT-ST', 'AI M' |
| btg | integer | jumlah batang (null jika tidak ada) |
| m3 | numeric(10,3) | kubikasi (null jika tidak ada) |

**Kode mutu yang valid:**

*Kayu Bernomor (KB):*
- `AIII_IS` — AIII IS/UT-ST
- `AIII_PD` — AIII P/D T/M
- `AIII_LN` — AIII L/N
- `AIII_KBP` — AIII KBP
- `CIII_IS` — CIII/CIV IS/UT-ST
- `CIII_PD` — CIII/CIV P/D/T/M
- `JML_KB` — Jumlah KB (sub-total)

*Kayu Tak Bernomor (TBN):*
- `AI_PD` — AI P/D/T
- `AI_M` — AI M
- `AI_KBP` — AI KBP
- `AII_UT` — AII UT/ST
- `AII_PD` — AII P/D/T
- `AII_M` — AII M
- `AII_KBP` — AII KBP
- `CI_IS` — CI IS/UT-ST
- `CI_PD` — CI P/D/T
- `CI_M` — CI M/L
- `CII` — CII
- `JML_TBN` — Jumlah TBN (sub-total)

## RLS Policies

- `dk310_periods`: operator SELECT/INSERT WHERE tpk_id = user's tpk_id; admin SELECT all
- `dk310_surat_bukti`: inherit via JOIN ke dk310_periods
- `dk310_surat_bukti_mutu`: inherit via JOIN ke dk310_surat_bukti → dk310_periods

---

Tambahkan di `App.jsx` dalam blok `ProtectedRoute`:


## Halaman Utama — `Dk310.jsx` (`/dk310`)

### Header
- Judul "DK310" + icon
- Tombol **Import Excel** (kanan atas) — hidden `<input type="file" accept=".xlsx,.xls">` dipicu oleh button, konsisten dengan pola DkhpSkshhk.jsx

### Summary Cards (5 cards)
Menampilkan **akumulasi total** dari semua periode yang tersimpan untuk TPK aktif:
1. Penambahan
2. Sisa yang Lalu
3. Jumlah Persediaan
4. Jumlah Pengurangan
5. Sisa Sekarang

Setiap card menampilkan dua nilai: **Btg** (teks putih) dan **m³** (teks hijau primer).

### Tabel Periode
Kolom: **No | Periode | Penambahan | Sisa Lalu | Jumlah Persediaan | Pengurangan | Sisa Sekarang**

Kolom data (selain No dan Periode) memiliki dua sub-kolom:
- **Btg** — teks normal/putih
- **m³** — teks warna hijau muted (`text-primary-400`)

Klik baris → navigate ke `/dk310/:id`.

---

## Halaman Detail — `Dk310Detail.jsx` (`/dk310/:id`)

### Header
- Tombol ← Kembali
- Judul periode (e.g. "DK310 — I Januari 2026")
- Info baris: KPH | TPK | Tanggal Cetak

### Summary Cards
Sama dengan halaman utama, tapi data dari periode ini saja.

### Tabel Surat Bukti
Kolom: **No | Nomor Surat | Jenis | Total Btg | Total m³**

Klik baris → **expandable row inline** menampilkan tabel rincian mutu di bawah baris tersebut.

### Expandable Row — Rincian Mutu
Tabel nested dengan kolom: **Mutu | Kategori | Btg | m³**

Dikelompokkan: KB di atas, TBN di bawah, dengan sub-total masing-masing.

## Excel Parser

File: `src/lib/parseDk310.js`

**Logika parsing:**
- Baris 1–9: metadata header (KPH, TPK, BKPH, masa pembayaran, tanggal cetak)
- Baris 11–187: data surat bukti dalam **pasangan baris** — baris ganjil = batang, baris genap = m³
  - Kolom mapping sesuai struktur Excel (col index 4=AIII_PD, 5=AIII_LN, 6=AIII_KBP, 9=JML_KB, 10=AI_PD, 11=AI_M, 12=AI_KBP, 13=AII_UT, 14=AII_PD, 15=AII_M, 16=AII_KBP, 17=CI_IS, 18=CI_PD, 19=CI_M, 20=CII, 21=JML_TBN, 22=JML_TOTAL)
  - Skip baris kosong (semua nilai null)
- Baris 188–201: summary (Penambahan, Sisa Lalu, Persediaan, Pengurangan, Sisa Sekarang) untuk masing-masing dk310_periods

**Output parser:** objek `{ periodData, suratBuktiList }` siap untuk di-insert ke Supabase.

Library: `xlsx` (sudah tersedia di project).

## Tema & Styling

Mengikuti tema Deskra yang sudah ada:
- Background gelap (`#0a0a0a`, `#0d0d0d`)
- Aksen hijau (`#379165`, `#00ff88`)
- Teks putih/off-white
- Card border dengan `border-white/10` atau `border-primary-500/20`
- Hover row: `hover:bg-white/5`
- Sub-kolom Btg: `text-white/90`, sub-kolom m³: `text-primary-400`
